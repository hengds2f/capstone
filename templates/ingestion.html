📖 What is Data Ingestion?

Data ingestion is the process of collecting, importing, and loading data from various sources into a storage system for analysis. In a data science workflow, this is the first step — without clean, reliable data, no analysis or model can succeed.

🎯 Business Purpose

In Singapore's context, data ingestion means pulling from sources like data.gov.sg, HDB, URA, LTA DataMall, and SingStat to build a unified data platform for urban analytics.

🔑 Key Concepts

  • Batch ingestion: Loading data at scheduled intervals (e.g., monthly HDB resale data)
  • Streaming ingestion: Continuous data flow (e.g., MRT crowding updates)
  • Schema validation: Checking data types, nulls, and constraints before loading
  • Idempotency: Running ingestion multiple times produces the same result

# Sample ingestion code used in this app
import pandas as pd
from models.database import get_connection

def ingest_csv_to_table(csv_path, table_name):
    """Load a CSV file into a SQLite table."""
    df = pd.read_csv(csv_path)

    # Validate before loading
    assert not df.empty, "DataFrame is empty"
    assert df.isnull().sum().sum() / df.size < 0.5, "Too many nulls"

    conn = get_connection()
    df.to_sql(table_name, conn, if_exists='append', index=False)
    conn.close()
    return len(df)

⚠️ Common Mistakes
  • Not validating schema before loading: Mismatched columns cause silent data corruption
  • Ignoring encoding issues: Singapore data often contains Chinese characters — use UTF-8
  • No deduplication: Running ingestion twice without checking creates duplicates
  • Hardcoding file paths: Use configuration files for production deployments
