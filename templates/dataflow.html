📖 Data Serialization Formats

Data must be serialized (converted to bytes) for storage and transmission. The format choice impacts performance, readability, and interoperability.

FormatTypeProsConsUse Case
CSVTextHuman-readable, universalNo schema, slow for big dataopen data portal downloads
JSONTextFlexible, API-friendlyVerbose, no native typesopen data portal API responses
ParquetBinaryColumnar, compressed, fastNot human-readableLarge-scale analytics
AvroBinarySchema evolution, row-basedComplex setupKafka event streams
Protocol BuffersBinaryCompact, fast, typedRequires .proto filesgRPC microservices
Python
import pandas as pd
import json

# CSV: Human-readable flat files
df = pd.read_csv('data/housing_data.csv')
df.to_csv('output/data_clean.csv', index=False)

# JSON: API-compatible structured data
data = df.head(5).to_dict(orient='records')
with open('output/data_sample.json', 'w') as f:
    json.dump(data, f, indent=2)

# Parquet: Columnar storage for analytics (10x smaller, 100x faster)
df.to_parquet('output/housing_data.parquet', engine='pyarrow')
df_parquet = pd.read_parquet('output/housing_data.parquet')

# Size comparison (typical for 1M rows):
# CSV: ~150 MB | JSON: ~250 MB | Parquet: ~15 MB

📖 Data Flow Patterns

Concept
Request-Response (Synchronous):
  Client ──request──> Server ──response──> Client
  Example: GET /api/housing → JSON response

Publish-Subscribe (Async):
  Producer ──event──> Message Queue ──event──> Consumer(s)
  Example: transit crowding event → Kafka topic → Dashboard + Alert service

Batch File Transfer:
  Source ──file──> Staging ──process──> Warehouse
  Example: Monthly housing CSV → Python ETL → SQLite star schema
