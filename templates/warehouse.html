📖 What is a Data Warehouse?

A data warehouse is a centralized repository optimized for analytical queries (OLAP). Unlike operational databases (OLTP), warehouses store historical, aggregated, and denormalized data.

🎯 Singapore Example

Our SQLite database acts as a mini data warehouse for Singapore's HDB and transport data, using a star schema design.

📖 Slowly Changing Dimensions (SCD)

When dimension data changes over time, we need strategies to handle history.

SCD TypeApproachSG Example
Type 1Overwrite old valueUpdate school address after relocation
Type 2Add new row with datesTrack MRT station name changes over time
Type 3Add column for old valueStore both old and new planning area names
SQL
-- SCD Type 2: Track historical changes
CREATE TABLE dim_station_history (
    station_id INTEGER,
    station_name VARCHAR(100),
    line_name VARCHAR(50),
    effective_from DATE,
    effective_to DATE,        -- NULL means current
    is_current INTEGER DEFAULT 1,
    PRIMARY KEY (station_id, effective_from)
);

-- Query: Get current station names
SELECT * FROM dim_station_history WHERE is_current = 1;

-- Query: Get station name as of a specific date
SELECT * FROM dim_station_history
WHERE effective_from <= '2020-01-01'
  AND (effective_to IS NULL OR effective_to > '2020-01-01');

📖 Aggregate Tables & Materialized Views

SQL
-- Pre-computed aggregation for dashboard performance
CREATE TABLE agg_monthly_prices AS
SELECT
    l.town,
    t.year,
    t.month,
    COUNT(*) as transaction_count,
    ROUND(AVG(f.resale_price), 0) as avg_price,
    MIN(f.resale_price) as min_price,
    MAX(f.resale_price) as max_price,
    ROUND(AVG(f.price_per_sqm), 2) as avg_price_sqm
FROM fact_hdb_transactions f
JOIN dim_location l ON f.location_id = l.location_id
JOIN dim_time t ON f.time_id = t.time_id
GROUP BY l.town, t.year, t.month;

-- Dashboard query now hits the aggregate instead of joining 3 tables
SELECT * FROM agg_monthly_prices WHERE town = 'BISHAN';
