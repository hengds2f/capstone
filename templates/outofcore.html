📖 When Data Exceeds Memory

When datasets are too large to fit in RAM, we process them in chunks. This is called "out-of-core" or "external memory" processing.

🎯 Real-World Scale

housing data from 1990-2024 contains ~900,000 records. While this fits in memory, real-world datasets from Transport Authority (taxi GPS, road toll) can exceed 100GB.

📖 Chunked CSV Processing

Python
import pandas as pd

# Process large CSV in chunks of 10,000 rows
chunk_results = []
for chunk in pd.read_csv('large_dataset.csv', chunksize=10000):
    # Filter within each chunk
    filtered = chunk[chunk['town'] == 'DOWNTOWN']
    # Aggregate within chunk
    summary = filtered.groupby('property_type')['price'].mean()
    chunk_results.append(summary)

# Combine all chunk results
final = pd.concat(chunk_results).groupby(level=0).mean()
print(final)

Generator Pattern for Memory Efficiency

Python
def process_large_file(filepath, batch_size=5000):
    """Generator that yields processed batches."""
    reader = pd.read_csv(filepath, chunksize=batch_size)
    for i, chunk in enumerate(reader):
        # Clean each chunk
        chunk['price'] = pd.to_numeric(chunk['price'], errors='coerce')
        chunk = chunk.dropna(subset=['price'])
        yield chunk
        print(f"Processed batch {i+1}: {len(chunk)} rows")

# Usage — memory never holds more than one batch
for batch in process_large_file('housing_data_full.csv'):
    batch.to_sql('raw_data', conn, if_exists='append', index=False)

📖 SQLite as Out-of-Core Engine

Python
import sqlite3
import pandas as pd

# SQLite handles data on disk — no need to load everything into RAM
conn = sqlite3.connect('datascience.db')

# Aggregate 1M+ rows without loading into memory
query = """
    SELECT town, property_type,
           COUNT(*) as count,
           AVG(price) as avg_price
    FROM raw_data
    GROUP BY town, property_type
    HAVING COUNT(*) > 100
    ORDER BY avg_price DESC
"""
result = pd.read_sql(query, conn)  # Only the result fits in RAM

📖 Memory-Mapped Files

Python
import numpy as np

# Create a memory-mapped array (stored on disk, accessed as if in RAM)
large_array = np.memmap('transport_data.dat', dtype='float32',
                        mode='w+', shape=(1000000, 10))

# Write data in chunks
for i in range(0, 1000000, 10000):
    large_array[i:i+10000] = np.random.randn(10000, 10)

# Access slices without loading entire array
subset = large_array[500000:500100]  # Only loads 100 rows
print(f"Mean of subset: {subset.mean():.4f}")
⚠️ Common Mistakes
  • Loading everything first: Don't call pd.read_csv() without chunksize on large files
  • Appending to a list forever: Aggregate within each chunk, don't store all chunks
  • Not using dtypes: Specify dtype={'price': 'float32'} to halve memory usage
