📖 Pipeline Patterns

Retry with Exponential Backoff

Python
import time, random

def retry_with_backoff(func, max_retries=3, base_delay=1):
    """Retry a function with exponential backoff + jitter."""
    for attempt in range(max_retries):
        try:
            return func()
        except Exception as e:
            if attempt == max_retries - 1:
                raise
            delay = base_delay * (2 ** attempt) + random.uniform(0, 1)
            print(f"Attempt {attempt+1} failed: {e}. Retrying in {delay:.1f}s")
            time.sleep(delay)

# Usage
retry_with_backoff(lambda: ingest_csv_to_db("hdb_resale.csv"))

Idempotent Pipeline Steps

Python
# BAD: Appends duplicates on re-run
def ingest_bad(df, table):
    df.to_sql(table, conn, if_exists='append')

# GOOD: Idempotent — safe to re-run
def ingest_good(df, table):
    conn.execute(f"DELETE FROM {table}")  # Clear first
    df.to_sql(table, conn, if_exists='append')

# BETTER: Upsert pattern
def ingest_upsert(df, table, key_col):
    for _, row in df.iterrows():
        conn.execute(f"""
            INSERT INTO {table} ({', '.join(row.index)})
            VALUES ({', '.join(['?']*len(row))})
            ON CONFLICT({key_col}) DO UPDATE SET
            {', '.join(f'{c}=excluded.{c}' for c in row.index if c != key_col)}
        """, tuple(row))

🏗️ Dagster Orchestration

Dagster is a modern data orchestrator built around Software-Defined Assets. Instead of defining "tasks that run in order," you declare what data should exist and Dagster figures out how to compute it.

🆚 Dagster vs Airflow vs Meltano

DAG definitionImperative PythonYAML configDeclarative assets
Data awareness❌ NoSinger schema✅ Full lineage

🔑 Key Dagster Concepts

  • @asset — A software-defined asset. Declare what data should exist; Dagster infers the DAG.
  • @op — A single computation unit (like an Airflow operator).
  • @graph / @job — A DAG of ops that can be executed together.
  • Resources — Shared services (DB connections, API clients) injected into ops.
  • IO Managers — Handle reading/writing between ops (e.g., SQLite, S3, Parquet).
  • Schedules — Cron-based triggers for jobs.
  • Sensors — Event-driven triggers (e.g., new file detected).

📖 Dagster Code Patterns

Software-Defined Assets

Python
from dagster import asset, AssetIn

@asset(group_name="raw")
def raw_hdb_resale():
    """Extract raw HDB data from CSV."""
    return pd.read_csv("data/hdb_resale.csv")

@asset(
    ins={"raw_hdb": AssetIn(key="raw_hdb_resale")},
    group_name="staging"
)
def hdb_price_stats(raw_hdb):
    """Compute price statistics per town."""
    return raw_hdb.groupby("town").agg(
        avg_price=("resale_price", "mean"),
        count=("resale_price", "count")
    ).reset_index()

# Dagster infers: raw_hdb_resale → hdb_price_stats

Ops, Graphs, and Jobs

Python
from dagster import op, graph, job, In, Out

@op(out=Out(io_manager_key="sqlite_io"))
def extract_csv(context):
    df = pd.read_csv("data/hdb_resale.csv")
    context.log.info(f"Extracted {len(df)} rows")
    return df

@op(ins={"df": In()}, out=Out(io_manager_key="sqlite_io"))
def transform_stats(context, df):
    result = df.groupby("town")["resale_price"].mean().reset_index()
    context.log.info(f"Transformed: {len(result)} towns")
    return result

@op
def validate(context, df):
    assert len(df) > 0, "Empty DataFrame!"
    context.log.info("Validation passed ✅")

@graph
def hdb_pipeline():
    raw = extract_csv()
    stats = transform_stats(raw)
    validate(stats)

# Convert to a runnable job with resources
hdb_job = hdb_pipeline.to_job(
    resource_defs={"sqlite_io": sqlite_io_manager}
)

Resources & IO Managers

Python
from dagster import IOManager, io_manager, resource
import sqlite3

class SQLiteIOManager(IOManager):
    def __init__(self, db_path):
        self.db_path = db_path

    def handle_output(self, context, obj):
        table = context.asset_key.path[-1]
        conn = sqlite3.connect(self.db_path)
        obj.to_sql(table, conn, if_exists="replace", index=False)
        conn.close()

    def load_input(self, context):
        table = context.asset_key.path[-1]
        conn = sqlite3.connect(self.db_path)
        return pd.read_sql(f"SELECT * FROM {table}", conn)

@io_manager(config_schema={"db_path": str})
def sqlite_io_manager(context):
    return SQLiteIOManager(context.resource_config["db_path"])

# Usage in Definitions
from dagster import Definitions
defs = Definitions(
    assets=[raw_hdb_resale, hdb_price_stats],
    resources={"io_manager": sqlite_io_manager.configured(
        {"db_path": "database/singapore_data.db"}
    )}
)

Schedules & Sensors

Python
from dagster import schedule, sensor, RunRequest, SkipReason

@schedule(cron_schedule="0 0 * * *", job=hdb_job)
def daily_hdb_schedule(context):
    return RunRequest(run_key=f"hdb-{context.scheduled_execution_time}")

@sensor(job=hdb_job, minimum_interval_seconds=60)
def new_csv_sensor(context):
    new_files = check_for_new_uploads()
    if new_files:
        yield RunRequest(
            run_key=f"upload-{new_files[0]}",
            run_config={"ops": {"extract": {"config": {"file": new_files[0]}}}}
        )
    else:
        yield SkipReason("No new files detected")
⚠️ Common Dagster Mistakes
  • Using ops when assets suffice: Most pipelines should use @asset. Only use @op for complex multi-output logic.
  • Hardcoding resources: Use resource injection so ops can be tested with mock resources.
  • Ignoring IO Managers: Let Dagster handle persistence — don't read/write files directly in ops.
  • No partition scheme: For time-series data, use partitioned assets for incremental processing.
  • Monolithic ops: Break large ops into smaller, testable units. Each op should do one thing.

🔧 Meltano ELT Pipeline

Meltano is an open-source DataOps platform for the modern data stack. It follows the ELT pattern (Extract → Load → Transform) using the Singer protocol for taps (extractors) and targets (loaders), with dbt for transformations.

🆚 ETL vs ELT

AspectETL (Traditional)ELT (Modern / Meltano)
Transform timingBefore loadingAfter loading (in-warehouse)
Data storedOnly transformed dataRaw + transformed data
FlexibilityFixed schema upfrontSchema-on-read, iterate fast
ToolsInformatica, SSIS, TalendMeltano, Airbyte, Fivetran + dbt

⚡ Run Meltano ELT

Execute ELT pipelines that extract from CSV sources, load raw data into SQLite, then run dbt-style SQL transforms to create staging/analytics tables.

📖 Meltano Configuration (meltano.yml)

YAML
version: 1
default_environment: dev

plugins:
  extractors:
    - name: tap-csv          # Singer tap for CSV files
      pip_url: git+https://github.com/MeltanoLabs/tap-csv.git

  loaders:
    - name: target-sqlite    # Singer target for SQLite
      pip_url: target-sqlite~=0.1
      config:
        database: database/singapore_data.db

  transformers:
    - name: dbt-sqlite       # dbt for SQL transformations
      pip_url: dbt-core dbt-sqlite

jobs:
  - name: hdb-elt
    tasks:
      - tap-csv target-sqlite   # Extract & Load
      - dbt-sqlite:run          # Transform

schedules:
  - name: daily-hdb-sync
    interval: "@daily"
    job: hdb-elt

📖 Singer Protocol & Meltano Architecture

Singer Messages

JSON
// SCHEMA message — declares the stream's schema
{"type": "SCHEMA", "stream": "hdb_resale",
 "schema": {"properties": {"town": {"type": "string"}, "resale_price": {"type": "number"}}}}

// RECORD message — a single data record
{"type": "RECORD", "stream": "hdb_resale",
 "record": {"town": "ANG MO KIO", "resale_price": 450000}}

// STATE message — checkpoint for incremental extraction
{"type": "STATE", "value": {"bookmarks": {"hdb_resale": {"last_extracted": "2024-01-15"}}}}

How This App Implements It

Python
from services.meltano_elt import MeltanoELTPipeline

# 1. Configure the pipeline
pipeline = MeltanoELTPipeline('hdb_elt')
pipeline.configure_extract('data/hdb_resale.csv', 'csv')
pipeline.configure_load('raw_hdb_resale')

# 2. Add dbt-style SQL transforms
pipeline.add_transform('stg_hdb_monthly_avg', """
    CREATE TABLE IF NOT EXISTS stg_hdb_monthly_avg AS
    SELECT month, town, flat_type,
           COUNT(*) as transaction_count,
           ROUND(AVG(resale_price), 2) as avg_price
    FROM raw_hdb_resale
    GROUP BY month, town, flat_type
""")

# 3. Run the full ELT
result = pipeline.run()
# result['steps'] → {'extract': {...}, 'load': {...}, 'transform': {...}}

⚠️ Common ELT Mistakes
  • Transforming before loading: ELT loads raw data first — don't filter or clean during extraction
  • Not using incremental loads: Full refreshes are expensive; use STATE bookmarks for incremental extraction
  • Skipping data validation: Raw data in the warehouse still needs quality checks (use great_expectations or dbt tests)
  • Monolithic transforms: Break dbt models into staging → intermediate → marts layers
